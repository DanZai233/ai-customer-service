# 知识库

## 职责

维护客服可引用的业务知识，提供草稿、发布、归档工作流，以及与 AI 回复一致的检索测试。

## 代码入口

- `src/app/(workspace)/knowledge/page.tsx`
- `src/components/knowledge/knowledge-workspace.tsx`
- `src/lib/knowledge/repository.ts`
- `src/lib/knowledge/search.ts`
- `src/app/api/knowledge`

## 当前实现

- 页面初始数据和统计由服务端从 PostgreSQL 读取。
- 创建和编辑通过受保护 API 持久化，不使用浏览器临时数组。
- `knowledge.write` 控制新建与编辑，`knowledge.publish` 控制发布与归档。
- 只有已发布文档进入后台检索测试和 AI 回复上下文。
- 检索测试显式触发，记录真实命中事件和文档命中次数。
- 支持标题/分类/正文筛选，以及草稿、已发布、已归档状态筛选。
- 行业模板可批量安装为知识草稿，并使用来源地址防止重复安装。
- 无写权限角色自动获得只读界面。

## 主要流程

1. 有写权限的用户新建草稿并编辑正文。
2. 有发布权限的用户发布草稿，文档立即进入检索。
3. 检索测试调用 `POST /api/knowledge/search`，返回最多三条命中及分数。
4. AI 回复使用相同仓储和排序逻辑构建组织级知识上下文。
5. 归档文档立即退出检索，但内容仍保留在数据库中。

## 验收

1. 新建草稿后刷新页面仍存在。
2. 草稿不出现在检索结果，发布后可以命中，归档后再次消失。
3. 编辑、发布和归档按钮按角色权限显示。
4. 运行检索后本月检索命中和文档命中次数增加。
5. 桌面与移动视口下表格、编辑抽屉和检索面板可正常操作。
